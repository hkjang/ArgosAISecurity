# Argos 정책 무결성 및 암호 서명 기능 분석서

**Argos 정책 무결성 통제 모듈**은 외부 침입자가 서버 에이전트의 차단/탐지 파라미터 설정을 임의 개조하여 탐지를 회피하려는 위협 시도를 차단하기 위해, 암호학적인 Ed25519 공개키 기반 서명 검증을 거친 화이트리스트 구성 정보만 인가하는 무결성 보호 솔루션입니다.

---

## 1. 핵심 설계 및 컴포넌트

정책 서명 검증은 [argos-policy](file:///d:/project/ArgosAISecurity/crates/argos-policy/src) 라이브러리가 전담합니다.
- **`Policy`** ([lib.rs](file:///d:/project/ArgosAISecurity/crates/argos-policy/src/lib.rs)): 외부 배포용 정책 데이터 형식으로 버전 값(`version`), 탐지 필터 구성(`DetectionConfig`), 대응 제어 파라미터(`ResponseConfig`)가 집약된 구조체입니다.
- **`ed25519_dalek`**: Ed25519 타원곡선 암호 디지털 서명(Edwards-curve Digital Signature Algorithm) 표준 규격을 지원하는 암호 라이브러리 엔진입니다.

---

## 2. Ed25519 기반 서명 및 정책 무결성 검증 원리

Argos는 정책 작성 시 설정 바이트 전체에 관리자 서명키를 서명해 안전한 무변조 증명을 확증합니다.

```
[관리 콘솔 측 작업]                     [대상 노드 서버 에이전트 측 검증]
정책 구성 (policy.toml) 작성             서명된 파일과 .sig 서명 해시 파일 로드
        │                                             │
        ▼                                             ▼
비밀 서명키(SigningKey) 로드                 설정에 심어진 공개키(VerifyingKey) 로드
        │                                             │
        ▼                                             ▼
정책 원본 파일 바이트 서명                    VerifyingKey::verify(바이트, 서명) 검사
        │                                             │
        ▼                                             ├─► 성공 ➔ load_verified 완료 및 세팅 교체
policy.toml.sig 서명 파일 생성                └─► 실패 ➔ InvalidSignature 거부 및 시동 시 세팅 고수
```

### 2.1. 서명 대상 정규화 회피 설계
- 구성 파일을 파싱하여 정렬화하는 가공 과정을 거치면 줄바꿈이나 공백 등 서식 차이로 인해 미세한 바이트 델타가 유발되어 올바른 서명 검증에 빈번히 실패하게 됩니다.
- Argos는 이러한 불안 요소를 원천 제거하고자 정책 설정 파일의 **원물 바이트 배열 전체를 정규화 과정 없이 100% 그대로 서명 및 대조**하는 단순하면서도 강력한 검증 설계를 적용했습니다.
- 단 1바이트의 공백 삽입이나 변조가 일어나도 암호 대조 단계(`verify_bytes`)에서 예외 없이 서명 대조 실패 에러(`PolicyError::InvalidSignature`)를 던지며 검증을 완벽히 방어합니다.

## 3. 에이전트 데몬 설정 연동 제어 로직

에이전트 데몬 서비스(`argos-agent`)가 기동될 때의 정책 유무 판독 및 교체 처리 과정은 다음과 같습니다:

1. **설정 확인**:
   - 로컬 구동용 설정 파일 `argos.toml` 내에 정책 경로(`policy.path`)와 신뢰 공개키(`policy.pubkey`) 항목이 명시되어 작동하고 있는지 헬스체크합니다.
2. **정책 파일 검증 로드 (`load_verified`)**:
   - `load_verified`는 배포 정책 파일(예: `policy.toml`) 및 서명 해시가 기입된 파일(예: `policy.toml.sig`)을 쌍으로 로드합니다.
   - hex 코드로 저장된 64자(32바이트) 검증용 공개키 문자열을 암호 검증키 개체(`VerifyingKey`)로 변환합니다.
   - `.sig`에 기록된 128자(64바이트)의 서명 hex 코드를 디코딩하여 서명 바이트(`Signature`)로 바꿉니다.
   - 공개키를 투입해 파일의 무결성 서명 확인 작업을 기동합니다.
3. **폴백 및 격리 처리**:
   - 만약 침입자가 임의로 정책 데이터 파일의 차단 및 탐지 옵션을 강제 해제하거나 백업 예외 디렉터리를 자신들의 거점으로 조작해 놓은 상태라면, 서명 불일치로 인해 검증이 차단됩니다.
   - 검증 에러 발생 시, 에이전트는 기형 정책의 메모리 적용을 즉각적으로 거절(Abort)하고 시동 시 가지고 있던 안전 지대의 자체 기본 설정을 강제로 고수(Fallback)하여 취약점이 유발되지 않도록 완벽히 제어합니다.
