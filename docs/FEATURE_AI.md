# Argos AI 분석 및 코파일럿 기능 분석서

**Argos AI 분석 및 코파일럿 모듈**은 감지된 로우 시스템 이벤트 로그(I/O, 프로세스 Exec)를 거대 언어 모델(LLM)에 주입하기 쉬운 형태의 문맥(Context)으로 결합하여, 사람이 쉽게 해독할 수 있는 한국어 침해 사건 레포트를 생성하고 대화형식의 자연어 코파일럿 감사를 수행하는 대화형 인텔리전스 분석 서브시스템입니다.

---

## 1. 핵심 설계 및 컴포넌트

AI 연동 및 자연어 질의는 [argos-brain](file:///d:/project/ArgosAISecurity/crates/argos-brain/src) 라이브러리가 전담합니다.
- **`ThreatExplainer`**: Anthropic Claude API와 연동을 제어하는 핵심 AI 래퍼 객체로, 사건 요약 설명(`explain`) 및 코파일럿 자연어 감사 대화(`ask`) 작업을 지원합니다.
- **`DetectionContext`**: 특정 침해 건에 대해 Claude API에 RAG로 결합하여 공급할 타겟 데이터 셋 구조체입니다. (탐지 룰, 위험 점수, 시각, PID, 타겟 파일 이력, 전후 윈도우 시간 범위 내의 원시 파일 시스템 로그 목록)
- **`CopilotContext`**: 호스트 전체 상태 정보 요약과 최근 80건의 원시 파일 시스템 로그, 최근 40건의 프로세스 기동 정보(cmdline)가 통합 축적된 RAG 대화 컨텍스트용 구조체입니다.

---

## 2. RAG 기반 데이터 수집 및 프롬프트 합성

인공지능의 지엽적인 상상력에 의존해 허위 원인이나 결과를 양산하는 **AI Hallucination(환각 현상)**은 EDR 분석 단계에서 중대 오판을 초래하므로 이를 엄격히 억제해야 합니다.

### 2.1. 컨텍스트 합성 절차
1. **Explain 요청 (특정 위협 분석)**:
   - 관리자가 `argos explain <ID>` 서브커맨드를 기동합니다.
   - CLI 도구는 해당 탐지 레코드의 SQLite 정보를 취출하여 발생 타임스탬프를 획득합니다.
   - 침해 유발 전 윈도우 시간 주기(기본 10초) 및 유발 후 5초 범위 내에서 가동된 모든 원시 파일 이벤트를 SQLite에서 덤프하여 `DetectionContext`로 패킹해 Claude에 주입합니다.
2. **Ask 요청 (자연어 코파일럿 감사)**:
   - 관리자가 `argos ask "지난 1시간 동안 의심 행위 있었어?"` 등으로 자연어 대화를 시동합니다.
   - CLI는 에이전트의 구동 세팅 상태 정보와 더불어 최근 DB에 저장된 최근 30건의 탐지, 최근 80건의 I/O, 최근 40건의 exec 명령어 기록 등을 일괄 덤프해 `CopilotContext`로 포장한 뒤 Claude에 주입합니다.

## 3. Anthropic Messages API 연동 및 시스템 통제 프롬프트

공식 Rust SDK가 존재하지 않는 개발 환경의 특성을 극복하기 위해 `reqwest` 동기식 요청 방식을 직접 래핑하여 Messages API 규격(`https://api.anthropic.com/v1/messages`)을 호출합니다.
- API 버전 헤더: `x-api-key`, `anthropic-version: 2023-06-01`
- 탑재 분석 모델: `claude-opus-4-8` (환경변수 `ARGOS_AI_MODEL`로 오버라이드 가능)

### 3.1. Hallucination 억제를 위한 시스템 프롬프트 제어
의사결정을 오도하는 거짓 분석 정보 유발을 방어하기 위해 다음과 같이 극도로 절제된 시스템 지침(`SYSTEM_PROMPT` 및 `COPILOT_SYSTEM`)을 Messages API 요청 시 하드코딩 형태로 제공합니다:

```
[SYSTEM PROMPT 지침 요약]
1. 오직 제공된 탐지 로그 및 파일/프로세스 실행 정보만을 토대로 한국어 분석을 실행할 것.
2. 로그 바깥의 주관적 정황 추론을 완전히 금지함.
3. 모든 분석 및 판단 결과에 관해, 근거가 되는 실측 이벤트 로그 라인(시각, 경로, PID)을 함께 제시할 것.
4. 확실하지 않은 상태나 근거 결여 상태는 반드시 "추가 확인 필요" 또는 "제공된 로그에서 확인되지 않음"으로 명시 보고할 것.
```

### 3.2. 한국어 정형 보고서 포맷팅 규격
Claude API가 처리한 결과는 관리자의 정밀 모니터링 편의를 위해 마크다운 규칙을 준수하는 다음 형태의 한국어 템플릿으로 출력됩니다:

- **`## 사고 요약`**: (비전문가도 즉시 식별 가능한 2~3줄 요약)
- **`## 근거 분석`**: (탐지를 직접 초래한 핵심 파일 쓰기 및 프로세스 명령어 패턴 분석)
- **`## 오탐 가능성`**: (정상 배치 실행 작업이나 백업, 로그 로테이션일 물리적 가능성과 대조 정황 근거 제시)
- **`## 권장 조치`**: (우선순위에 맞춘 격리, 프로세스 롤백 권고 가이드 수록)
